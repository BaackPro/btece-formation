Étapes pour configurer Netlify avec GitHub :
Préparer votre dépôt GitHub

Créez un nouveau dépôt sur GitHub

Initialisez localement :

bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/votre-utilisateur/btece-formation.git
git push -u origin main
Configuration Netlify

Connectez-vous à Netlify

Choisissez "New site from Git"

Sélectionnez GitHub comme provider

Autorisez l'accès à votre dépôt

Choisissez le dépôt btece-formation

Configuration du build (netlify.toml)

toml
[build]
  command = "echo 'No build needed'"
  publish = "src"  # Ou "." si votre HTML est à la racine

[context.production.environment]
  NODE_ENV = "production"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
Variables d'environnement (si besoin)

Dans Netlify : Site settings > Build & deploy > Environment

Ajoutez :

NODE_ENV = production

Domain personnalisé (optionnel)

Dans Netlify : Domain settings

Ajoutez votre domaine (ex: formation.btece.com)

Bonnes pratiques :
Séparation des ressources :

Extrayez le CSS dans un fichier séparé

Déplacez les scripts JS vers assets/js/

Optimisation des images :

Compressez les images avant de les ajouter

Utilisez le format WebP quand possible

Continuous Deployment :

Netlify déploiera automatiquement à chaque git push

Activez "Deploy previews" pour les PR

Fonctionnalités avancées (optionnelles) :

Ajoutez un formulaire Netlify pour gérer les inscriptions

Activez les fonctions serverless si besoin

Configurez des redirections dans netlify.toml

Pour extraire le CSS (recommandé) :
Créez assets/css/style.css et déplacez-y tout le contenu entre <style> et </style>

Dans votre HTML, remplacez par :

html
<link rel="stylesheet" href="/assets/css/style.css">
Cette structure vous donnera :

Un déploiement continu via GitHub

Une gestion simple des mises à jour

Une bonne organisation du code

La possibilité d'ajouter facilement des fonctionnalités
